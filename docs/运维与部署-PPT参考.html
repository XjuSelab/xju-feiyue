S13

部署 · 生产架构

一台机器，两个角色：nginx 既是门面也是网关，systemd 让后端永不掉线。

整套平台只跑在一台华为云 VPS 上。取巧之处在于让一台机器同时演好两个角色——nginx 一边把前端构建产物当静态资源直出，一边把 /api 反代到后端；后端交给 systemd 托管，崩了自动拉起。发布则被压成一条脚本，最后用一次心跳探活确认「真的活着」。

华为云 VPS · huawei2winbeau.top 主 + feiyue.selab.top 镜像 systemd · feiyue-backend :8001nginx 静态服 + 反代
整个平台跑在一台华为云 VPS 上。nginx 同时做两件事：把前端 dist 作静态资源直接发出去，同时把 /api 反代到后端；后端是 FastAPI，用 uvicorn 跑在 8001 端口，由 systemd 托管，开机自启、崩溃自动拉起。上线只要一条 deploy.sh：拉主分支、同步依赖、迁移数据库、重启服务、健康探活、再构建前端。winbeau.top 是主域名，feiyue.selab.top 是镜像。
S14

部署 · CI 与双库维护

五个人协作不乱套：双库分工 + 提交即质检。

五个人一起写，代码怎么不打架？我们立了两条规矩：一个库专管规范和文档、一个库专管真正在生产上跑的代码，职责互不越界；而每一次提交都先过一道本地质检门，规范不达标直接拦下，再进后面的发布流水线。

　团队规范库 · XjuSelab/xju-feiyue部署源 · winbeau/Aurash
职责文档 / 规范 / 源码评审生产实际运行的代码
下游GitHub Pages「发展历程」时间轴自动部署deploy.sh 拉取 → 华为云上线
门禁conventional commits（feat/fix/docs…）+ Husky + lint-staged 提交前自动检查
我们用双库策略：一个团队规范库放文档和规范，一个部署源跑生产代码，职责分离。所有提交走 conventional commits，Husky 配 lint-staged 在提交前自动检查、拦下不规范的改动，历史干净可追溯。对外的「发展历程」时间轴页由 GitHub Pages 自动部署。前端用 pnpm + Vite 打包，后端用 uv 锁死依赖版本，换机也一致。
S15

运维 · 数据持久化与恢复

数据是运维的命根子：异地留存，一条命令换机复现。

运维里我最当回事的一件事——数据绝不能只躺在这一台机器上。所以数据库、用户上传的附件、连同密钥，都定时同步到一个私有的 HuggingFace 数据集异地留存，还按命名空间把「实时状态」和「参考数据」分开放。真要换机或翻车，一条命令就能把整站数据拉回来复现。

make sync-push # 定时(cron)把 DB+附件+密钥推到私有数据集
make data-pull # 换机 / 灾备：一键拉回整站数据复现
make schools-pull / conf-pull # 拉取高校导师 / CCF 会议参考库
运维里最要命的是数据不能丢。我把数据库、上传附件和密钥全部同步到一个私有的 HuggingFace 数据集异地留存，里面按命名空间分区：state 放实时状态，schools 和 conf 放高校导师库和会议库。日常备份由 make sync-push 定时 cron 完成；一旦换机或灾备，一条 make data-pull 就能把整站数据拉回来复现。内容真源是 62 篇 seed，加上站内用户直接发布的笔记。
S16

运维 · 安全与性能

两条线并行：安全四道防线守住「稳」，两个预算守住「快」。

🛡 安全 层层设防

重点是双重防存储型 XSS：上传时按扩展名拦掉 svg/html，静态服务再对每个响应补 nosniff 头，即使有漏网也不会被浏览器当脚本执行。

⚡ 性能 预算约束

首屏 JS 压缩后 ≈144KB、全路由懒加载；AI 欢迎语缓存 3h 轮换省下大量 DeepSeek 调用。WritePage 因编辑器+diff+公式偏大，已诚实标为已知偏差。

🗄 管理后台第四道保障：登录事件审计 · 数据热 reload（不停机换数据）· 会议爬取手动触发。 〔可插一张后台截图，我访问不了运行中的系统，需你自截〕
安全和性能上我做了四道防线。鉴权用 HS256 的 JWT，7 天有效，密码 bcrypt 哈希。文件安全上，上传拦掉 svg、html 这类能被当脚本执行的扩展名，静态服务再统一注入 nosniff 头，双重防存储型 XSS，另加 50MB 上限和 magic-byte 校验。性能上首屏 JS 压缩后约 144KB，全路由懒加载，AI 欢迎语缓存 3 小时省 API。管理后台能审计登录、热更新数据、手动触发会议爬取。
