## 1. 系统总览

xju-feiyue 是面向新疆大学学生/校友的社区式知识沉淀平台，核心是 7 分类（research/course/recommend/competition/kaggle/tools/life）的长文 Markdown 笔记（草稿→发布、点赞、锚点引用评论），并围绕它扩展出五个独立 feature 纵切：**共享课程资料库**（递归文件树 + 在线 PDF/Word/Excel 预览 + 拖拽排序）、**CCF 会议截稿追踪器**（~230 条，72h 自动爬取）、**导师/院校招生情报库**（C9+，外部工具导出）、**教务成绩单一键导入的学分自查**（浏览器扩展 + pdf.js 客户端解析）、以及 **DeepSeek 驱动的个性化问候与 AI 笔记润色**。后端是一个 FastAPI 单体，前挂三套物理隔离的存储域：一个可写的主库（用户/笔记/草稿/资料）与两个只读、可热重载的外部导出 SQLite（schools / conferences）。

## 3. 组件职责

### 3.1 后端 API / 路由层（`backend/app/`，FastAPI 单体）
FastAPI 应用工厂 + DI + 11 个 router，共暴露 60 个路由装饰器（远超文档宣称的 19）。
- **`main.py`** — App 工厂、lifespan（建 uploads 目录、boot schools/conferences engine holder、拉起 author-sync 与 conference-crawl 后台循环、teardown）、CORS、`/uploads` 静态挂载用 `HardenedStaticFiles`（强制 `X-Content-Type-Options: nosniff`）、router 注册顺序、`/health`。
- **`deps.py`** — DI 核心：`get_db`、`client_ip`（信任自家 nginx 的 `X-Forwarded-For` 最左 / `X-Real-IP`）、`get_current_user`（401）、`get_optional_user`（静默 None，用于公开+个性化端点）、`require_admin`（**非管理员返回 404 使整个 /admin 面不可发现**）、`require_superadmin`（普通 admin 403、非 admin 404）。
- **`settings.py`** — pydantic-settings：`database_url`、jwt、DeepSeek 凭证、`cors_origins`、`public_base_url`（拼到存储的头像/上传 URL 前）、`admin_sid`（**永远解析为 superadmin，无视 DB role 列**）、schools/conferences 数据目录、author_sync / conf_crawl 开关与间隔。
- **`schemas/_base.py`** — `CamelModel`（全站 camelCase，`populate_by_name` 兼收 snake_case，`from_attributes`）与 `SnakeModel`（仅 `/schools/*` 用）；`UtcDateTime` 恒输出带尾 `Z` 的 ISO-8601。
- **routes 一览**：`auth.py`（login/logout/me/password/avatar，PIL 缩略、2MB 限、格式白名单）· `admin.py`（3 级角色仪表盘：用户列表聚合查询、stats 含 14 天上海时区登录火花线、创建用户默认密码 123456、重置密码、角色升降、登录审计；并 re-export `require_admin`/`require_superadmin` 供 schools/conferences 导入）· `notes.py`（列表 filter/cursor 分页、hot/latest/liked、get?id=、PATCH 作者自限并按需 AI 重摘要、DELETE 级联；**无 POST create**）· `drafts.py`（全 CRUD + `/publish` 校验并 AI 摘要，Draft→Note）· `interactions.py`（点赞幂等 upsert、评论 keyset 分页、评论删除作者双向授权）· `ai.py`（`/ai/compose` 无鉴权、`/ai/greetings` 3h 缓存降级本地模板、`/ai/compose/stream` SSE 先吐注释行破 nginx 缓冲、错误进 `{error}` 事件）· `materials.py`（**共享**知识库文件树，读公开、写需 `owner_sid==user.sid`；上传冲突自动改名；下载 RFC5987 Content-Disposition+nosniff；notice 单例栏）· `schools.py`（snake_case 域，offset 分页，跑独立只读引擎叠加主库 UGC，未同步则 503）· `conferences.py`（list + reload + 手动 crawl）· `transcript.py`（**未鉴权** `/notes/transcript-stash` 中转，内存 dict 5 分钟 TTL 10MB PDF 魔术字节校验；GET 需鉴权取走即删）· `uploads.py`（`/notes/images` 8MB、`/notes/files` 50MB，挂 `/notes/*` 前缀纯为落进 nginx allowlist）。

### 3.2 后端数据层（`backend/app/db/` + `backend/alembic/`）
- **`base.py`** — `Base(DeclarativeBase)`；全局 `Engine.connect` 监听器对每个 SQLite 连接发 `PRAGMA foreign_keys=ON`（否则 CASCADE 在 SQLite 静默失效）。
- **`models.py`** — 主库唯一 schema 源：User/Note/Draft/Like/Comment/LoginEvent/MaterialResource/MaterialFile/MaterialNotice；`StringList`（PG `ARRAY(Text)` / SQLite `JSON`）、`CATEGORY_VALUES`（7 分类做普通字符串列而非原生枚举，保 SQLite 测试廉价）。
- **`session.py`** — 进程级 async engine + `AsyncSessionLocal`（`expire_on_commit=False`, `autoflush=False`）+ `get_session()` 依赖。
- **`schools_engine.py` / `conferences_engine.py`** — 单例 holder，包只读 `AsyncEngine`（`mode=ro&immutable=1&uri=true` + `PRAGMA query_only=ON` 纵深防御）；每次 `get_engine()` 轮询文件 mtime，变了就换新引擎、dispose 旧的；缓存 sibling `manifest.json`；文件缺失抛 `*DataMissing` → 路由译成 503，不影响 app 启动。
- **`alembic/env.py`** — async 环境，强制用 `settings.database_url` 覆盖 `alembic.ini`。
- 迁移链严格线性 `0001..0009`，见第 4 节。

### 3.3 后端服务层（`backend/app/services/`）
薄路由与存储/外部服务之间的业务缝：
- **`ai_compose.py`** — DeepSeek 客户端；`build_prompt()` 映射 7 模式（polish/shorten/expand/tone/translate/custom/summarize）；`compose()` 同步单次调用后走 `diff.compute_diff_segments`；`stream_chunks()` SSE 异步生成器；`summarize_or_fallback()` 永不抛错；`deepseek_dry_run` 全局 kill-switch（echo/假 chunk）。
- **`greeting.py`** — 每用户 3 条问候，3h 进程内缓存；`familiar_name()` 从法定名推称呼；7 时段本地模板兜底；open-meteo 天气**只传定性描述不传数值**；最多 `min(6,2n)` 次 DeepSeek 调用 + `is_valid_greeting()` 校验去重；任何异常静默降级。
- **`diff.py`** — diff-match-patch 薄封装，与前端 `diffEngine.ts` 算法一致。
- **`auth.py`** — bcrypt + HS256 + 3 级角色（`admin_sid` 恒 superadmin）。
- **`comments.py`** — 真 keyset 分页（`created_at,id` 元组比较，over-fetch+1）。
- **`conferences_crawler.py`** — conferences.sqlite 生产者：ccfddl allconf/allacc.yml 主源（缩写匹配，置信 0.95 无 LLM）、DDG+主页抓取+DeepSeek JSON 抽取兜底；`crawl_sync()` 拷到临时文件、`sqlite3` 直写、`os.replace()` 原子换、重写 manifest；按 deadline 派生 crawl_state（unannounced/announced/closed）与 next_check_at（+1d/+5d/None）。
- **`conferences_query.py`** — 全表 dump（~230 行，前端过滤）。
- **`materials.py`** — 文件树从单条 flat SELECT + Python 分组装配（避开 `lazy='raise'` 的 MissingGreenlet）；`clean_name()`（不做 markdown 转义，纯文本树节点）、`unique_upload_name()`、级联软删+物理 unlink、`reorder_file()` 含祖先环检测。
- **`notes.py`** — `summary_from()`、`read_minutes_from()`（~500 CJK 字/分）、批量点赞/评论计数、`list_notes()`（全量载入后 Python 排序/tag 过滤/cursor 分页，明确标注仅适合数千量级）。
- **`schools_overlay.py`** — **UGC 叠加桩**，硬编码 `note_count=0`/`is_starred=False`（依赖表未建）。
- **`schools_pinyin.py`** — 内存拼音模糊索引（pypinyin，随 sqlite mtime 重建）。
- **`schools_query.py`** — 全部 filter/sort/paginate/FTS5 下推 SQLite；`_build_where()` OR 合并 FTS5 MATCH + 转义 LIKE + 拼音 id 集；含防注入转义。
- **`uploads_common.py`** — 共享上传原语：1MiB 分块流式写、超限前中止、扩展白名单 + `DENY_EXTS`（.svg/.html/.htm/.xml 防存储型 XSS）、`sniff_magic()` 魔术字节交叉校验、服务端定 MIME、`safe_display_name()` 转义 markdown 元字符。
- **`author_sync.py`** — 每日 `repair()` 以 `content/notes/*.md` frontmatter 为真值校准 `notes.author_sid`，只改 DB 不改 md。

### 3.4 前端核心/基建（`frontend/src/`）
- **`vite.config.ts`** — Vite+Vitest，`@`→src，vitest jsdom 仅扫 `src/**/*.test.*`（排除 Playwright）。
- **`main.tsx`/`App.tsx`** — QueryClientProvider（staleTime 30s、retry 1、关窗口聚焦刷新）+ RouterProvider + sonner Toaster；挂载即 `authStore.hydrateFromToken()`。
- **`router.tsx`** — 全路由 lazy+Suspense（PageBoundary/RouteError）；`/login` 在 AppShell 之外；`/admin`、`/demo` **无导航入口仅 URL 可达**；`/_dev/design-system` 仅 DEV。
- **`api/client.ts`** — 唯一 `request<T>()` fetcher，按 `useMock = DEV && !VITE_API_BASE` 分支；每个响应无论真假都 `schema.parse()`；`resolveAssetUrl()` 把绝对上传 URL 重写到当前 API origin 以规避跨域。
- **`api/index.ts`** — 数据层 barrel，仅 `DEV && !VITE_API_BASE` 时副作用导入 mock；导出所有端点命名空间与全部 TanStack hook（useNotes 无限滚动、useToggleLike 乐观、useComments 等）。
- **`api/endpoints/*.ts`** — 每资源一文件；各自重复实现 `authHeaders()` 读 `localStorage['labnotes.auth.token']`（**重复 ~8 次未集中**）；`ai.ts` 含 `composeStream()` 手写 SSE 解析 + 3 层兜底。
- **`api/mock/*`** — 内存 mock 分发；**仅** auth/notes(list/hot/latest/liked/get)/ai/notes-images/schools/conferences 有 handler，drafts/interactions/materials/admin/notes-files **无 mock，pure mock 下抛 501**。
- **`api/schemas/*.ts`** — Zod 层，每资源一文件带 `.test.ts`；递归 `z.lazy()` 文件树；204 用 `NoContentSchema`。
- **`api/upload.ts`** — `xhrUpload()` 用 XHR（fetch 无上传进度事件）。
- **`stores/authStore.ts`** — zustand+persist，key `labnotes.auth`，mode `authed|guest|anon`；login 顺带预取 greetings；`hydrateFromToken()` 用 `/auth/me` 复验。
- **`stores/draftStore.ts`** — 纯客户端草稿（key `labnotes.drafts`，versioned migrate 补 summary），**与后端 Drafts CRUD 是两个不同概念**。
- **`components/layout/*`** — AppShell/Header/MegaMenu/Footer/`RequireAccess`（mode 表驱动路由守卫）/GitHubStars（裸 fetch GitHub API）。

### 3.6 浏览器扩展 / 用户脚本（`extension/` + 前端 credits 部分）
- **`extension/content.js` + `manifest.json`** — MV3 content script（`world:MAIN`），仅注入 `jwxt-443.webvpn.xju.edu.cn`（按钮）与 feiyue/winbeau.top（自报）。点击后：`POST ?method=topdf` → 从返回文件名 `<sid>_<ts>.pdf` 解析 sid（**不读 cookie，因 webvpn_username 常 HttpOnly**）→ `GET ?method=download` 取 PDF blob → `no-cors` multipart POST 到 `/notes/transcript-stash`。守 `window.top!==window.self`（jwxt 是 frameset）与 `window.__feiyueImporter` 防重入。

## 4. 数据模型

### 4.1 主库 ORM 表与关系图（`backend/app/db/models.py`）
`users.sid`（String(11)，11 位学号，自然主键）是枢纽，所有归属 FK 都 `ondelete='CASCADE'` 指向它：

```
                         users (sid PK, name, nickname, preferred_name,
                                avatar, avatar_thumb, bio, wechat, phone,
                                email, password_hash, role, created_at)
        ┌──────────┬─────────────┬──────────────┬───────────────┬────────────┐
        ▼          ▼             ▼              ▼               ▼            ▼
   notes      drafts        likes         comments       login_events  material_
   author_sid owner_sid  user_sid(复合PK) author_sid      user_sid       resources
   (id PK)    (id PK)    +note_id(复合PK)  (id PK)         (id自增PK)      owner_sid
      │                      │               │                            (id PK)
      │ id                   │ note_id       │ note_id                       │ resource_id
      ▼                      ▼               ▼                               ▼
   ┌────────── notes.id 被 likes/comments 引用(均 CASCADE) ────┐    material_files
   │ 删 Note → DB 级联删 Likes/Comments                        │    (id PK, parent_id
   └───────────────────────────────────────────────────────────┘    自引用CASCADE树,
                                                                      is_folder, ext, mime,
   material_notice: 单例(id='default' 固定, content, visible,        size_bytes, url,
                    updated_by_sid 无FK纯审计, updated_at)            storage_path, sort_order,
                                                                      deleted 软删)
```

要点：
- **点赞**：复合主键 `(note_id, user_sid)` + 一条冗余 `UniqueConstraint uq_likes_note_user`（与主键功能重复，无害）。
- **草稿** `drafts.id` 独立，**不 FK 关联最终 Note**（发布时新建 Note、删 Draft）。
- **资料树**：`material_files.parent_id` 自引用（根为 NULL）；`(resource_id, parent_id)` 内名字唯一性**在服务层强制而非 DB 约束**（因 SQLite 视 NULL parent_id 两两不等，真约束挡不住根级重名）。
- **删除策略**：全靠 DB 级 `ondelete='CASCADE'` + ORM `passive_deletes=True`，一对多关系几乎全 `lazy='raise'`（禁 async 隐式懒加载），**从不用** SQLAlchemy `cascade='all, delete-orphan'`——树列表端点必须从 flat SELECT 手搭响应对象（否则 pydantic `from_attributes` 触 `lazy='raise'` 引发 MissingGreenlet）。
- `login_events.id` 是唯一的自增整型主键（其余皆 String uuid/自然键）。

### 4.2 三引擎拆分与理由
| 引擎 | 文件 | 读写 | 内容 | 为什么拆 |
|---|---|---|---|---|
| 主库 | `labnotes.db` | 读写 ORM+Alembic | 用户/笔记/草稿/点赞/评论/登录/资料 | 事务性 UGC，须随 `state/` HF 镜像备份 |
| schools | `data/schools/schools.sqlite` | 只读（mode=ro&immutable=1 + query_only） | 4000+ 导师行（advisor/appointment/school/…），**schema 不在 models.py/Alembic** | 外部 supervisor-claw 离线导出，大（~20MB，git push 超时故 gitignore），可再生 |
| conferences | `data/conferences/conferences.sqlite` | 只读引擎读 / 爬虫 `sqlite3` 直写 | ~230 CCF 会议 + crawl_state | ccfddl/DeepSeek 爬虫再生，同样 gitignore+HF 同步 |

两只读引擎共用**轮询 mtime 热重载**（每次 `get_engine()` 比对 mtime，变则加 `asyncio.Lock` 换新引擎 dispose 旧的）+ `force_reload()` 逃生口（`/admin/{schools,conferences}/reload` 与爬虫收尾后调用）。写者永远"临时文件 → `os.replace()` 原子换 → 重写 manifest"，读者绝不见半成品。**schools/conferences 的表结构对 Alembic 与 `Base.metadata` 完全不可见**——只能从服务层 raw SQL 字符串反推，权威契约在外部 claw 仓库的 `EXPORT_SCHEMA_v1.md`。

### 4.3 迁移演进（`backend/alembic/versions/`，严格线性 0001→0009）
- **0001** init：users/notes/drafts/likes/comments。
- **0002** `users.avatar_thumb`（160px 缩略；回填靠 `scripts/backfill_avatar_thumbs.py`，**不在迁移内**）。
- **0003** `login_events`（管理员审计，IP 来自 XFF）。
- **0004** comments 加 `anchor_text/anchor_offset_start/anchor_offset_end`（选段引用，MVP 仅渲染 anchor_text）。
- **0005** `drafts.summary`（`server_default=''`）。
- **0006** materials（`material_resources` + 自引用 `material_files` 树；docstring 注明"autogenerate 后手改命名 + down_revision 手钉"）。
- **0007** `users.preferred_name`（称呼，NULL 运行时经 `greeting.familiar_name` 派生）。

## 5. 关键数据流（端到端，标注真实文件）

### (a) 用户登录 / 鉴权
1. 前端 `LoginForm`（react-hook-form + zodResolver）→ `api/endpoints/auth.ts login()` → `client.ts request()` → `POST /auth/login {sid,password}`。
2. `routes/auth.py` login：SELECT User by sid → `services/auth.py verify_password`（bcrypt）→ `create_access_token(sid)`（HS256，`{sub:sid,iat,exp}`，7 天）。
3. 插入 `LoginEvent`（`deps.client_ip()` 取 XFF 最左 + 截断 UA）。
4. 返回 `LoginOut{user, token}`；前端 `stores/authStore.ts` persist 到 `localStorage['labnotes.auth']`，并 fire-and-forget 预取 `/ai/greetings` 写 3h 缓存。
5. 此后每请求带 `Authorization: Bearer <token>`（各 `endpoints/*.ts` 各自 `authHeaders()` 读 `labnotes.auth.token`）。
6. `App.tsx` 挂载调 `authStore.hydrateFromToken()` → `GET /auth/me` 复验，失败落 `anon`；`deps.get_current_user` 解签验期 + `db.get(User,sid)`，任一失败 401。

### (b) 笔记发布 + AI 润色（DeepSeek streaming diff）
1. 编辑 `WritePage.tsx`，内容经 `hooks/useAutoSave.ts` 存入 `stores/draftStore.ts`（本地）；同时可走 `api/endpoints/drafts.ts` 建后端 Draft。
2. 划选 ≥4 字，`MarkdownEditor.tsx` 报选区矩形 → `FloatingToolbar` 出润色；选模式 → `ai/useAICompose.ts` 一次性 `POST /ai/compose` → `routes/ai.py` → `services/ai_compose.py compose()`（DeepSeek，temp 0.3，max 2000）→ `services/diff.py` 出 add/del/equal 段。
3. `ai/diffEngine.ts computeDiff` 前端同算法呈现 `DiffView`；**accept/reject 仅整块**（`applyAccept/applyReject` 过滤全部段），无逐段。
4. 摘要生成走真流式：`SummaryField.tsx` → `useSummaryCompose.ts` → `api/endpoints/ai.ts composeStream()` → `POST /ai/compose/stream`（SSE，先吐 `: ready` 破 nginx 缓冲，`X-Accel-Buffering:no`）→ `stream_chunks()`（DeepSeek stream，max 300），逐 chunk 实时打字进摘要框；**3 层兜底**：mock 合成 / 非 OK 回退一次性 / 5s 无首字节回退一次性重放。
5. 发布 `POST /notes/drafts/{id}/publish`（`routes/drafts.py`）：校验 title/content/category，摘要空则 `ai_compose.summarize_or_fallback`（DeepSeek 失败降级首段启发式）；新建 Note（id=uuid4，read_minutes 按长度算）、删 Draft，一次 commit。

### (c) 教务成绩单导入（扩展 → transcript-stash → /credits）
1. 用户在浏览器登录 `jwxt-443.webvpn.xju.edu.cn`（真 session cookie，扩展从不见密码）。
2. `extension/content.js`（或托管 userscript / bookmarklet，逻辑等价）点击：`POST ?method=topdf`（credentials:'include'）→ 从 `j.result` 解 `<sid>_<ts>.pdf` 路径（sid 匹配 `/guest/i` 视为未登录）。
3. `GET ?method=download&fileSavePath=...` → PDF blob。
4. `no-cors` multipart POST 到 `https://feiyue.selab.top/notes/transcript-stash`（fire-and-forget）→ `routes/transcript.py stash_transcript()` 验 `%PDF-` + ≤10MB，存内存 `dict[sid]=(bytes, now+300s)`。
5. `/credits` 页 `features/credits/lib/stash.ts fetchStashedTranscript`（visibilitychange/focus + 每 2.5s 主动轮询，180s 超时）`GET /notes/transcript-stash`（Bearer）→ 后端 pop 即删本人 sid 条目（204 若无）。
6. 同一 `parseTranscript.ts` + `rules.ts` 客户端 pdf.js 解析出学分报告（**PDF 全程不落服务器**）。
> 存在原因：feiyue 与 jwxt 跨源，CAS 的 COOP 在 WebVPN 登录重定向后切断 `window.opener`，postMessage 不可靠；故绕道未鉴权按 sid、5 分钟 TTL、内存 stash，且挂在既有 `/notes` nginx location 内免改反代。

### (d) 资料上传与预览
1. 任意登录用户 `POST /materials/resources` 建自有资源；`POST /materials/resources/{rid}/files`（owner-only，multipart）。
2. `services/uploads_common.py save_upload`：扩展白名单 + `DENY_EXTS`、1MiB 分块流式（超 50MB 前中止）、`sniff_magic()` 魔术字节交叉校验、服务端定 MIME；`services/materials.py unique_upload_name` 冲突自动 `(n)`，流到 `uploads/materials/<sid>/<rid>/`。
3. `MaterialFile` 行追加 `sort_order`；树由单条 flat SELECT + `build_file_tree` 重建。
4. 读：`GET /materials/resources[/{rid}[/files]]`（optional-auth，**返回所有人非删资源——共享库**）。
5. 前端 `features/materials/components/preview/PreviewPane.tsx` 按扩展懒加载 `PdfViewer/DocxViewer/ExcelViewer/ImageViewer/CodeViewer`（各自裸 `fetch(resolveAssetUrl(url))` 而非走 client.ts），ErrorBoundary 降级下载卡；下载走 `useDownload.ts`（fetch+ReadableStream 保同源 blob + 真进度）。
6. 拖拽：`lib/tree.ts projectDrop` 逐帧算落点，`useReorder` onMutate 乐观改缓存（`applyReorder` 镜像后端 `reorder_file` 语义）、onSettled invalidate、onError 回滚。

### (e) CCF 会议自动爬取
1. `main.py` lifespan 起 `_conf_crawl_loop`（每 `conf_crawl_interval_hours` 默认 72h，首轮 `full_scan=True`）；`routes/conferences.py POST /admin/conferences/crawl` 亦可手动。
2. `services/conferences_crawler.py crawl_sync()`（`asyncio.to_thread`）：拷 sqlite 到临时文件；取 ccfddl `allconf.yml`+`allacc.yml`（2 GET）；选 due 行（`crawl_state!='closed' AND next_check_at<=now`，或 full_scan 全表）。
3. 逐会议：ccfddl 匹配优先（置信 0.95 无 LLM）；否则 `ddgs` 搜 + httpx 抓主页 + stdlib 剥 script/style + DeepSeek JSON 抽取（temp 0），会间 1s 礼貌 sleep。
4. 按 deadline 派生 crawl_state 与 next_check_at（+1d/+5d/None），`COALESCE` UPDATE 只覆盖抽到的字段。
5. commit → `os.replace()` 原子换 → 重写 manifest（sha256+计数）→ 若有变更 `conf_holder.force_reload()`，`conferences_engine.py` 侦测 mtime 换新只读引擎，下个请求即见新数据。

### (f) 内容 Notion → markdown → 站点
1. `scripts/notion_pull.py`：43 个硬编码 Notion 页 UUID 经公开 `loadPageChunk` API → `notion_raw/<slug>.json`（缓存可重跑）。
2. `scripts/notion_to_md.py`：纯 recordMap 遍历（无 LLM）→ `content/notes/<slug>.md.body`（GFM，标题降一级，code/quote/callout/table/toggle/equation 映射）。
3. `scripts/wrap_frontmatter.py`：读 `_index.json`+`.md.body`，关键词 `TAG_RULES` 派生 tags、首段启发式摘要、字数/300 算 readMinutes → 写 `content/notes/<slug>.md` 并删 `.md.body`。
4. 并行 `scripts/xju_pull.py`→`xju_to_md.py`→`xju_wrap.py`（17 课程 wiki，author 孙海洋，交错时间戳）。
5. `backend/scripts/seed.py`（drop+recreate → 入库 Note，硬编码 2 账号 USERS 解析 author→sid，未知默认 winbeau）；`scripts/derive_mock_notes.py`（4 账号 AUTHOR_MAP → 前端 mock notes.json）；`services/author_sync.py` 每日校准 `author_sid` 回 frontmatter。
> `content/notes/` 共 62 文件 = 43 Notion + 17 xju-wiki + 2 手写 baoyan-*；生产 85 篇的差额来自站上直接发布的 DB-only 笔记。

## 6. 前后端契约

- **数据层铁律**：Component → TanStack Query hook（`api/index.ts`）→ `api/endpoints/<resource>.ts`（**必须 Zod `.parse()`**）→ `api/client.ts request<T>()`（唯一 fetch 封装）。（注：SSE 流、文件预览/下载、GitHubStars 等 ~15 处已是合法的直 fetch 例外，`INTEGRATION_REPORT` 宣称"fetch 只 1 处"已过时。）
- **mock↔真后端切换**：`useMock = import.meta.env.DEV && !VITE_API_BASE`，`client.ts` 与 `api/index.ts` 动态 import 同一判据。**只需写 `VITE_API_BASE=...` 到 `.env.local` 即切真后端，无需改源/注释导入**（`BACKEND_SPEC §6` 的"注释掉一行 import"已是 no-op）。mock 覆盖不全：drafts/interactions/materials/admin/notes-files 无 handler，pure mock 下抛 501——**本地开发大部分功能已需真后端**。
- **baseURL** 默认 `''`（同源直穿，prod nginx 按顶层路径前缀 auth/notes/drafts/interactions/ai/health/uploads/admin 反代），**非** 文档所说 `/api`；`VITE_API_BASE` 仅跨源 dev/staging 需要。
- **wire format**：全站 camelCase（`CamelModel`，`populate_by_name` 兼收 snake_case），**唯一例外** `/schools/*` 用 snake_case（`SnakeModel`，因 claw 导出与前端 `schools/types.ts` 本就 snake_case）。时间恒带尾 `Z`。
- **分页两制**：站点主流 cursor（`nextCursor`，`/notes` 线性 id 扫描、评论真 keyset）；**但 `/schools/list` 用 offset `page/page_size`**（`PaginatedAdvisors{items,total,page,page_size}`）——未文档化的第二套约定。
- **鉴权**：`Authorization: Bearer <JWT>`；`RequireAccess` 前端按 `mode` 表同步放行/跳 `/login`；`/admin` 双层守卫（router 只查 requireAuth，真 admin 判定在 AdminPage 内 mirror 404-for-non-admin，无导航入口）。
- **204** 前端用 `NoContentSchema = z.null()`；上传 `ApiError({detail})` 形状在 `client.ts`/`upload.ts` 统一，`AbortSignal` 语义一致。

## 7. 值得注意的设计决策与安全措施

- **Bootstrap superadmin 身份制而非纯 DB flag**：`settings.admin_sid` 在 `services/auth.py effective_role()` 恒解析 superadmin，无视 role 列——迁移出错/列滞后也锁不死运维（0008 回填仅为仪表盘显示一致）。
- **`require_admin` 对非管理员返 404**（非 401/403）使整个 `/admin` 面**不可发现**；`require_superadmin` 对普通 admin 升 403（他已知面存在）仍对非 admin 404。
- **上传防存储型 XSS 两层**：①应用层 `uploads_common.DENY_EXTS` 上传时挡 .svg/.html/.htm/.xml；②`HardenedStaticFiles` 对每个 `/uploads` 响应加 `nosniff`。更强的 doc 类 `Content-Disposition:attachment` **刻意推到 prod nginx**（免破头像/PDF 内联预览），是"二选一"的运维保证（在 main.py 与 uploads_common.py docstring 声明，需 huawei2 runbook 落实，**仓库内不可验证**）。魔术字节交叉校验（PDF/OOXML-via-zipfile/OLE2/PNG/GIF/JPEG/WEBP）防改名可执行文件；服务端定 MIME 不信客户端。
- **资料库是有意的共享知识库**：读全公开/optional-auth 返所有人非删资源，仅改动需 `owner_sid==user.sid`。
- **笔记无 POST create**：唯一发布路径是 Draft→publish，AI 摘要兜底逻辑集中在此。
- **只读 sqlite 三重防御** + 轮询 mtime 热重载 + 缺失降级 503 不崩全局。
- **transcript-stash 故意未鉴权**（jwxt 页无 feiyue JWT，且 COOP 断 opener）：按 sid、内存、5 分钟 TTL、取即删、10MB、PDF 魔术字节；sid 从文件名解析（不读 HttpOnly cookie），`/guest/i` 视未登录。
- **AI 永不硬失败用户体验**：greetings 降级本地时段模板（永不 503），发布/编辑摘要降级首段，`/ai/compose/stream` 异常进 `{error}` SSE 事件不破 HTTP 状态。`deepseek_dry_run` 全局 kill-switch 让测试零联网；仅 `/ai/compose(/stream)` 处上游失败仍以 502/504 上抛。
- **问候天气只传定性描述**（剥温度/坐标）防模型复述硬数据。
- **CodeMirror 本地 patch**（`patches/@codemirror__view.patch` + `ime-patch.test.ts` 读 dist 断言）：开桌面 EditContext、修破坏中文 IME 的正则——真实活跃痛点，文档未提。
- **前端刻意重复后端业务逻辑**：`lib/tree.ts` 拖拽投影镜像 Python `reorder_file`（双语双份 + 双侧环检测）；`isValidGreeting`/`familiarName` 镜像后端；diff 算法两端一致。
- **materials `lazy='raise'` + flat SELECT 手搭响应**：防 async 隐式懒加载 MissingGreenlet（plan-materials 明列为具体生产故障模式）。
- **拷贝/下载工程细节**：上传用 XHR（fetch 无上传进度），下载用 fetch+ReadableStream（跨源 `<a download>` 被浏览器静默忽略）；`resolveAssetUrl()` 把绝对上传 URL 重写同源避 CORS 挂死。
- **同步/部署防脚**：三命名空间共一 HF Dataset，`delete_patterns` 各限前缀互不清除；deploy `--pull-data` 绝不拉 `state/`（否则回滚生产 DB）；health-check `--noproxy 127.0.0.1` 绕 GFW 代理假失败；cron 烘焙代理环境变量。
