## 1 引言

### 1.1 编写目的

本说明书面向后端与前端的维护工程师、代码审查者及后续接手者，对 xju-feiyue 平台**类/模块级**的内部设计做代码级还原：给出程序系统的组织结构、每个核心设计类的职责与方法签名、复杂控制流的过程设计（PDL/状态迁移），以及若干关键算法的实现契约。与《概要设计说明书》相比，本文下沉到**方法签名、参数类型、分支条件与不变式**一级，是把代码事实固化为设计文档的最贴地一层。

### 1.2 范围

覆盖后端 FastAPI 单体（`backend/app/`）中承担实际业务逻辑的服务层与关键控制器，以及与后端**镜像**的前端算法（diff、文件树投影、学分校验）。不含纯 UI 组件排布、样式令牌、内容离线管线（`scripts/`）的细节——后者见对应专项文书。

### 1.3 设计基线与关键事实

| 事实 | 依据（真实文件） |
|---|---|
| 用户自然主键为 11 位学号 `users.sid`，JWT 的 `sub` 即 sid | `services/auth.py:36` `create_access_token(sid)` |
| 鉴权采用 HS256 JWT + bcrypt，令牌有效期由 `jwt_expire_minutes` 控制（基线：7 天） | `services/auth.py:17,36-43` |

### 2.2 调用层次（分层与依赖方向）

严格单向依赖 `routes → services → db/schemas`，控制器不含业务分支，服务层不 import 路由：

```
        HTTP 请求
           │
           ▼
   ┌───────────────┐   Depends(...)   ┌─────────────────────────┐
   │  routes/*.py  │◀────────────────▶│  deps.py（DI：会话/鉴权） │
   │ （薄控制器）  │                  └─────────────────────────┘
   └──────┬────────┘
          │ 调用
          ▼
   ┌───────────────┐        ┌──────────────────────────────┐
   │  services/*   │───────▶│ 外部：DeepSeek / ccfddl / DDG  │
   │ （业务逻辑）  │        └──────────────────────────────┘
   └──────┬────────┘
          │ ORM / raw SQL
          ▼
   ┌───────────────────────────────────────────────────────┐
   │ db/：主库 AsyncEngine · schools 只读引擎 · conf 只读引擎  │
   └───────────────────────────────────────────────────────┘
```

三个典型调用链：

- **草稿发布**：`routes/drafts.py::publish` → `services/notes.py::summary_from/read_minutes_from` +（摘要空时）`services/ai_compose.py::summarize_or_fallback` → `services/diff.py`（内部不用）→ 主库 commit。
- **AI 润色**：`routes/ai.py::compose` → `services/ai_compose.py::compose` → `services/diff.py::compute_diff_segments`。
- **会议爬取**：`main.py` lifespan `_conf_crawl_loop` /（手动）`routes/conferences.py` → `services/conferences_crawler.py::crawl_sync`（`asyncio.to_thread`）→ `db/conferences_engine.py::force_reload`。

## 3 核心设计类

对每个类/模块给出：**职责 · 关键属性 · 方法签名（含参数与返回类型）· 依赖与协作者**。所有签名均摘自真实源文件并标注路径行号。

### 3.1 鉴权服务 `services/auth.py`

**职责**：密码散列与校验（bcrypt）、访问令牌签发与解码（HS256 JWT）、三级授权档位判定。这是理解全站所有权限门的钥匙。

**关键常量**（`auth.py:17-22`）：`ALGO = "HS256"`；`ROLE_USER="user"` / `ROLE_ADMIN="admin"` / `ROLE_SUPERADMIN="superadmin"`。

**方法签名**：

| 签名 | 位置 | 语义 |
|---|---|---|
| `hash_password(plain: str) -> str` | `auth.py:25` | `bcrypt.hashpw(...gensalt())` |
| `verify_password(plain: str, hashed: str) -> bool` | `auth.py:29` | `bcrypt.checkpw`，`ValueError` 吞为 `False`（防坏散列崩溃） |
| `create_access_token(sid: str) -> str` | `auth.py:36` | 载荷 `{sub:sid, iat, exp}`，`exp = now + jwt_expire_minutes` |
| `decode_token(token: str) -> str \| None` | `auth.py:46` | 验签验期；失败/`sub` 非 str 返回 `None`（不抛） |
| `effective_role(user: "User") -> str` | `auth.py:61` | **实际生效档位**：`user.sid==settings.admin_sid or role==SUPERADMIN` → superadmin；否则按 role 列，未知降级 user |
| `is_superadmin(user) -> bool` | `auth.py:76` | `effective_role(user)==SUPERADMIN` |
| `is_admin(user) -> bool` | `auth.py:80` | `effective_role(user) in {admin, superadmin}`（/admin-capable 集合） |

**关键不变式**：`effective_role` 使 `admin_sid` **无视 DB `role` 列**恒为 superadmin——迁移出错或列滞后也锁不死运维（Alembic 0008 回填仅为仪表盘显示一致，非授权真值）。

**依赖与协作者**：依赖 `app.settings.settings`（`jwt_secret`/`jwt_expire_minutes`/`admin_sid`）、`bcrypt`、`jose.jwt`。协作者：`deps.py` 的 4 个鉴权依赖、`services/materials.py::ensure_owner`（调用 `is_admin` 放行管理员 CRUD-any）。

### 3.2 权限依赖 `deps.py`

**职责**：FastAPI DI 层，产出数据库会话、客户端 IP、当前用户与三档权限门。全部为 async 依赖函数（`require_admin`/`require_superadmin` 表现为门卫）。

**方法签名**：

| 签名 | 位置 | 失败语义 |
|---|---|---|
| `get_db() -> AsyncGenerator[AsyncSession, None]` | `deps.py:13` | `async with AsyncSessionLocal()` |
| `client_ip(request: Request) -> str` | `deps.py:18` | XFF 最左 → X-Real-IP → `request.client.host`；信任自家 nginx |
| `get_current_user(request, db) -> User` | `deps.py:37` | 无 Bearer/无效 token/用户不存在 → **401** |
| `get_optional_user(request, db) -> User \| None` | `deps.py:50` | 同上但**静默 None**（公开+个性化端点用，如 `likedByMe`） |
| `require_admin(user=Depends(get_current_user)) -> User` | `deps.py:65` | 非 admin → **404**（`/admin` 面不可发现） |
| `require_superadmin(user=...) -> User` | `deps.py:76` | 非 admin → 404；普通 admin → **403**；superadmin 放行 |

**关键设计**：`require_admin` 与 `require_superadmin` 的**差异化状态码**（404 vs 403）是刻意的信息泄露最小化——普通 admin 已知面存在故返 403，非 admin 连面的存在都不该知道故返 404（`deps.py:70-72,81-86` docstring 明述）。

**协作者**：`decode_token`/`is_admin`/`is_superadmin`（来自 3.1）、`AsyncSessionLocal`（`db/session.py`）、`User`（`db/models.py`）。被所有需鉴权的路由 `Depends`。

### 3.3 草稿发布控制器 `routes/drafts.py`

**职责**：Draft 全 CRUD + `/publish`（草稿→笔记的唯一发布路径）。控制器层，业务下沉到 `services/notes.py` 与 `services/ai_compose.py`。前缀 `/notes/drafts`，全部 `Depends(get_current_user)` 强鉴权。

**私有归属守卫**（`drafts.py:21-27`）：
```
_get_owned_draft(draft_id: str, user: User, db: AsyncSession) -> Draft
    draft = await db.get(Draft, draft_id)
    if not draft or draft.owner_sid != user.sid: raise 404「草稿不存在」
```
> 归属校验与"不存在"合并返 404，不泄露"存在但非你所有"。

**路由方法签名**：

| 方法 | 装饰器 | 签名 | 出参 |
|---|---|---|---|
| 创建 | `POST ""` 201 | `create(body: DraftIn, user, db) -> DraftOut` | 新建 `Draft(id=uuid4, owner_sid=user.sid, ...)` |
| 列表 | `GET ""` | `list_mine(user, db) -> list[DraftOut]` | 仅本人，按 `updated_at desc` |
| 详情 | `GET /{draft_id}` | `get_one(draft_id, user, db) -> DraftOut` | 经归属守卫 |
| 更新 | `PATCH /{draft_id}` | `update(draft_id, body: DraftIn, user, db) -> DraftOut` | 逐字段 `is not None` 判定（**空串合法覆盖**，`None` 才跳过） |
| 删除 | `DELETE /{draft_id}` 204 | `delete(draft_id, user, db) -> Response` | 硬删 |
| **发布** | `POST /{draft_id}/publish` | `publish(draft_id, user, db) -> NoteOut` | 见 4.1 |

**发布关键契约**（`drafts.py:108-159`）：三段 422 校验（title/content/category 缺失）；`summary` 为空则调 `ai_compose.summarize_or_fallback(draft.content, summary_from(draft.content), title=draft.title)`；新建 `Note(id=uuid4, read_minutes=read_minutes_from(content), author_sid=user.sid)`、`db.delete(draft)`、**一次 commit**（发布与删草稿原子）；手工构造 `NoteOut`（`likes=0/comments=0`，`author=NoteAuthorOut(sid,nickname,avatar)`）避免触 `lazy="raise"`。

**协作者**：`Draft`/`Note`/`User`（models）、`DraftIn`/`DraftOut`/`NoteOut`/`NoteAuthorOut`（schemas）、`ai_compose.summarize_or_fallback`、`notes.summary_from`/`read_minutes_from`。

### 3.4 AI 组合服务 `services/ai_compose.py`

**职责**：把前端 7 种模式翻译成 DeepSeek 系统提示词，同步/流式调用上游，并把结果 diff 成 add/del/equal 段。永不让 AI 拖垮用户体验（摘要兜底、dry-run kill-switch）。

**关键属性**：模块级单例 `_client: AsyncOpenAI | None`（惰性构造）；上游模型 `settings.deepseek_model`（基线 `deepseek-v4-flash`）；`settings.deepseek_dry_run` 全局开关。

**方法签名**：

| 签名 | 位置 | 语义 |
|---|---|---|
| `get_client() -> AsyncOpenAI` | `:24` | 惰性单例，`base_url` 指向 DeepSeek，带 `timeout` |
| `build_prompt(mode: str, options: dict \| None) -> str` | `:35` | 映射 7 模式；未知模式 `raise ValueError` |
| `compose(req: AIComposeIn) -> AIComposeOut` | `:83` | 单次调用（temp 0.3，max 2000）→ `compute_diff_segments` |
| `stream_chunks(req: AIComposeIn) -> AsyncIterator[str]` | `:121` | 流式（max 300），逐 delta yield 裸文本（无 SSE 帧） |
| `summarize_or_fallback(content, fallback, *, title=None) -> str` | `:161` | **永不抛错**：dry-run/空内容/上游失败均返 `fallback` |

**7 模式映射**（`build_prompt`，`:37-80`）：`polish`（保留 markdown 语法润色）/`shorten`（压到 60-70%）/`expand`（扩到 1.5-2×）/`tone`（`options.target` formal/casual）/`translate`（`options.target` zh/en）/`custom`（`options.prompt` 自定义）/`summarize`（≤35 中文字符一句话简介，传 `title` 时**引导避免复述标题**）。

**上游错误映射**（`compose`/`stream_chunks` 共用，`:102-107,153-158`）：`RateLimitError`→**502**「AI 服务繁忙」；`APIStatusError`→502；`APIConnectionError`/`APIError`→**504**「上游超时或不可达」。`deepseek_dry_run` 时 `compose` 回显输入、`stream_chunks` 把输入切 5 段每段 sleep 0.12s（供 dev/测试看流式 UX，零联网）。

**协作者**：`compute_diff_segments`（`services/diff.py`）、`AIComposeIn/Out`（`schemas/ai.py`）、`openai.AsyncOpenAI`。被 `routes/ai.py`（compose/stream）与 `routes/drafts.py::publish`（summarize）调用。

### 3.6 资料树服务 `services/materials.py`

**职责**：`/materials/*` 域业务缝。铁律——**绝不遍历 `lazy="raise"` 关系**（`.files`/`.children`/`.parent`/`.resource`），文件树一律"单条 flat SELECT + 纯 Python 分组装配"，杜绝生产 `MissingGreenlet`。含名字唯一性守卫、级联软删+物理 unlink、带环检测的 reorder、owner 权限门。

**关键常量**：`UPLOAD_ROOT = <repo>/backend/uploads`；`MATERIALS_DIR = UPLOAD_ROOT/materials`；`_MAX_NAME_LEN=200`。磁盘布局 `uploads/materials/<sid>/<rid>/<fname>`。

## 4 过程设计

对 4 个复杂控制流用结构化伪代码（PDL）与状态迁移还原。

### 4.1 草稿 → 发布流程（含 AI 摘要兜底）

对应 `routes/drafts.py::publish`（`:108-159`）+ `ai_compose.summarize_or_fallback`（`:161-181`）。

```
PROC publish(draft_id, user, db) -> NoteOut
  draft ← _get_owned_draft(draft_id, user, db)      // 不存在或非本人 → 404

  // ── 三段发布前校验（缺一即 422，附中文 detail）──
  IF draft.title.strip() == ""    THEN raise 422「发布前必须填写标题」
  IF draft.content.strip() == ""  THEN raise 422「发布前必须填写正文」
  IF NOT draft.category           THEN raise 422「发布前必须选择分类」

  // ── AI 摘要兜底（用户没写简介时）──
  summary ← draft.summary.strip()
  IF summary == "" THEN
      // 默认走 AI；DeepSeek 不可达则降级"首段启发式"
      summary ← await ai_compose.summarize_or_fallback(
                    draft.content,
                    fallback = summary_from(draft.content),   // 首段截 200
                    title    = draft.title)
  ENDIF

  // ── 构造 Note、删 Draft，一次 commit（原子）──
  note ← Note(id=uuid4(), title, summary, content, category,
              tags=list(draft.tags), author_sid=user.sid,
              created_at=now_utc(), read_minutes=read_minutes_from(content))
  db.add(note); await db.delete(draft); await db.commit(); await db.refresh(note)

  // 手工构造出参，避免触 lazy="raise"（不读 note.author 关系）
  RETURN NoteOut(id, title, summary, cover, category, tags,
                 author=NoteAuthorOut(sid=user.sid, nickname, avatar),
                 created_at, likes=0, comments=0, read_minutes)
ENDPROC

PROC summarize_or_fallback(content, fallback, *, title) -> str
  IF content.strip()=="" OR settings.deepseek_dry_run THEN RETURN fallback
  TRY
      out ← await compose(AIComposeIn(mode="summarize", text=content,
                                      options={"title":title} if title else None))
      RETURN out.after.strip() OR fallback     // 上游空回也退 fallback
  CATCH HTTPException:                          // 502/504 一律吞
      RETURN fallback
ENDPROC
```

**不变式**：无论 DeepSeek 是否可达，`publish` 都成功——摘要最坏降级为正文首段。发布与删草稿在同一事务，不会出现"草稿已删但笔记未建"的中间态。

### 4.2 AI 润色 compose → diff 段生成

对应 `routes/ai.py::compose`（`:16-18`）→ `ai_compose.compose`（`:83-118`）→ `diff.compute_diff_segments`（`:15-19`）。注意 `/ai/compose` **无鉴权**（`routes/ai.py` 该路由未挂 `Depends(get_current_user)`）。

```
PROC compose(req: AIComposeIn) -> AIComposeOut
  started ← monotonic()
  sys_prompt ← build_prompt(req.mode, req.options)   // 未知模式 ValueError→422

  IF settings.deepseek_dry_run THEN
      after ← req.text                               // 回显（离线/测试）
  ELSE
      TRY
          resp ← client.chat.completions.create(
                     model, messages=[system(sys_prompt), user(req.text)],
                     temperature=0.3, max_tokens=2000)
      CATCH RateLimitError  → raise 502
      CATCH APIStatusError  → raise 502
      CATCH APIConnectionError|APIError → raise 504
      after ← resp.choices[0].message.content OR ""
  ENDIF

  elapsed_ms ← int((monotonic()-started)*1000)
  RETURN AIComposeOut(
      segments = compute_diff_segments(req.text, after),   // 见下
      before   = req.text, after = after, elapsed_ms)
ENDPROC

PROC compute_diff_segments(before, after) -> list[DiffSegment]
  diffs ← diff_match_patch().diff_main(before, after)      // char-level
  diff_cleanupSemantic(diffs)                              // 合并成语义块
  RETURN [ DiffSegment(type=_DMP_OP[op], text) FOR (op,text) IN diffs IF text≠"" ]
ENDPROC
```

**契约**：`segments` 形状与前端 `DiffSegment` Zod 完全一致（`type∈{equal,add,del}, text`）；前端 `diffEngine.ts::computeDiff` 用同算法本地重算呈现 `DiffView`，accept/reject **仅整块**（`applyAccept` 滤除全部 del、`applyReject` 滤除全部 add），无逐段。

### 4.3 会议爬虫 crawl_sync 的临时文件 → 原子替换 → 热重载

对应 `conferences_crawler.py::crawl_sync`（`:309-472`）。

```
PROC crawl_sync(data_dir, *, api_key, base_url, model, limit, dry_run, full_scan) -> dict
  sqlite_path ← data_dir/"conferences.sqlite"; manifest_path ← data_dir/"manifest.json"
  IF NOT sqlite_path.exists() THEN RETURN {error:"...not found", updated:0}
  today ← date.today(); now_iso ← utcnow "…Z"

  // ── 1. 主源：两次 GET 覆盖 ~95% ──
  allconf ← _fetch_yaml(ALLCONF) or []; allacc ← _fetch_yaml(ALLACC) or []
  ccfddl_idx ← _build_ccfddl_index(allconf); acc_idx ← _build_acc_index(allacc)

  // ── 2. 拷到临时文件后操作（读者永远看不到半成品）──
  (fd, tmp_path) ← tempfile.mkstemp(suffix=".sqlite", dir=data_dir); os.close(fd)
  shutil.copy2(sqlite_path, tmp_path); conn ← sqlite3.connect(tmp_path)

  IF full_scan THEN
      rows ← SELECT ... FROM conferences ORDER BY rowid
  ELSE
      rows ← SELECT ... WHERE crawl_state!='closed'
                        AND next_check_at IS NOT NULL AND next_check_at<=now_iso
                    ORDER BY next_check_at ASC          // 到期行
  IF limit THEN 追加 LIMIT limit
  due ← rows
  IF due==∅        THEN conn.close(); unlink(tmp); RETURN {due:0,...}
  IF dry_run       THEN conn.close(); unlink(tmp); RETURN {due:len, dry_run:true}

  // ── 3. 逐会议：ccfddl 优先，否则 DDG+DeepSeek ──
  ds_client ← OpenAI(...) IF api_key ELSE None
  FOR c IN due:
      target ← c.target_year OR today.year
      m ← _find_ccfddl_match(c.abbr, ccfddl_idx)
      IF m THEN
          extracted ← _extract_from_ccfddl(m, _find_ccfddl_match(c.abbr, acc_idx), target)  // 置信 0.95
      ELIF ds_client THEN
          page ← _fetch_page(c.homepage) OR _search_and_fetch(c.abbr, c.name_full, target)
                     OR "(No info… set found=false)"
          extracted ← _ask_deepseek(ds_client, model, c, page)
          time.sleep(INTER_CALL_SLEEP)               // 1s 礼貌
      ENDIF

      IF NOT extracted OR NOT extracted.found THEN
          UPDATE conferences SET last_checked_at=now, next_check_at=_next_check(c.crawl_state, now) WHERE id=c.id
          CONTINUE                                    // 未抓到也推进 next_check_at
      ENDIF

      deadline ← extracted.deadline OR c.deadline
      new_state ← _derive_state(deadline, today)
      UPDATE conferences SET <字段>=COALESCE(?, <字段>), …,   // 只覆盖抽到的字段
             crawl_state=new_state, last_checked_at=now,
             next_check_at=_next_check(new_state, now) WHERE id=c.id
  ENDFOR

  conn.commit(); count ← SELECT COUNT(*); conn.close()

  // ── 4. 原子替换 + 重写 manifest ──
  os.replace(tmp_path, sqlite_path)                   // 同盘 rename，原子
  raw ← sqlite_path.read_bytes()
  manifest_path.write_text(json{schema_version, exported_at, claw_version,
                                sha256=hashlib.sha256(raw), bytes, counts})
  RETURN {due, updated, found, ccfddl_matched, deepseek_used, results}
ENDPROC
```

**热重载衔接**：`os.replace` 是同一文件系统内的原子 rename，`conferences.sqlite` 的 mtime 随之跳变；`db/conferences_engine.py` 的只读引擎每次 `get_engine()` 轮询 mtime，检测到变化即加锁换新引擎、dispose 旧的（写完还可 `force_reload()` 主动触发）。因此并发读者要么看到旧库、要么看到新库，**绝不见半成品临时文件**。

### 4.4 会议 crawl_state 状态机与 next_check_at 派生

对应 `_derive_state`（`:290-296`）与 `_next_check`（`:299-304`）。以 `deadline` 与 `today` 为输入的三态机：

```
                    deadline 解析失败 / 无 deadline
        ┌──────────────────────────────────────────────┐
        ▼                                                │
  ┌────────────┐   抓到未来 deadline    ┌────────────┐   │
  │ unannounced│──────────────────────▶│  announced │   │
  │(未公布截稿)│                        │(已公布截稿)│   │
  └─────┬──────┘◀──────────────────────└──────┬─────┘   │
        │        deadline 又变无效/清空          │        │
        │                                        │ 到期(deadline<today)
        │  next=+1d                    next=+5d  ▼
        │                                 ┌────────────┐
        └────────────────────────────────│   closed   │  next_check_at=None
                       到期                └────────────┘  （不再复查）
```

**状态派生规则** `_derive_state(deadline, today)`：

| 条件 | 结果 state |
|---|---|
| `deadline is None` | `unannounced` |
| `date.fromisoformat(deadline)` 抛 `ValueError` | `unannounced` |
| `deadline < today` | `closed` |
| 否则（未来截稿） | `announced` |

**下次检查派生** `_next_check(state, now_iso)`：

| state | next_check_at |
|---|---|
| `closed` | `None`（终态，不复查） |
| `unannounced` | `now + UNANNOUNCED_DAYS(=1d)`（盯得紧，等公布） |
| `announced` | `now + ANNOUNCED_DAYS(=5d)`（已知截稿，稀疏复查） |

**选行谓词耦合**（4.3 第 2 步）：非 `full_scan` 扫描仅取 `crawl_state!='closed' AND next_check_at<=now` 的到期行——终态 `closed`（`next_check_at=None`）被自动排除，形成"派生 next → 下轮据 next 选行"的自洽闭环。

## 5 关键算法

### 5.1 diff-match-patch 前后端镜像

**目标**：AI 润色的增删可视化，后端出段、前端也能本地重算，两端结果逐字节一致。

- **后端** `services/diff.py::compute_diff_segments`（`:15-19`）：`diff_main(before, after)` 得字符级 diff → `diff_cleanupSemantic` 合并语义块 → 按 `_DMP_OP={-1:del,0:equal,1:add}` 映射 → 过滤空 text 段。
- **前端** `editor/ai/diffEngine.ts::computeDiff`（`:14-22`）：`new diff_match_patch().diff_main` + `diff_cleanupSemantic`，`op===INSERT?'add':op===DELETE?'del':'equal'`。

**镜像不变式**：两端同一算法库（Python `diff_match_patch` / npm `diff-match-patch`）、同一清理步骤、同一 op 三值域。对中文（每字符独立）与英文（自然词）都得到可读粒度。前端 `applyAccept`（滤 del）/`applyReject`（滤 add）实现**整块**接受/拒绝——`before` 由 equal+del 还原、`after` 由 equal+add 还原。

### 5.2 资料树 reorder 环检测（双侧镜像）

**目标**：拖拽移动文件/文件夹时，禁止把文件夹拖进自身或其子孙（否则形成环，树遍历死循环）。

**后端** `services/materials.py::reorder_file`（`:416-483`）算法：

```
1. 取 drag、drop（各 get_file_or_404）
2. 前置校验：同 resource（否则 400）；drag≠drop（否则 400）
3. rows ← list_resource_files(rid); by_id ← {id:row}
4. 解析新父域：
     position=='inside' → drop 必须是文件夹（否则 400），new_parent_id=drop.id
     before|after       → new_parent_id=drop.parent_id
5. ── 祖先环检测 ──
     cursor ← new_parent_id
     WHILE cursor is not None:
         IF cursor == drag.id: raise 400「不能把文件夹移动到自身或其子目录下」
         cursor ← by_id[cursor].parent_id      // 沿父链上爬
6. siblings ← 新父域内、排除 drag 的有序兄弟
     插入位置：inside→末尾；before→drop 索引；after→drop 索引+1
7. siblings.insert(index, drag); drag.parent_id=new_parent_id
8. 重写整域 sort_order = 0..n（原子）
```

**前端镜像** `materials/lib/tree.ts`：`applyReorder`（`:330`）+ `isDescendantOrSelf`（`:454-467`，`while cur: if cur===ancestorId return true; cur=parentId`）沿父链上溯做**同一环路守卫**（`:433` 注释明述"对齐后端 reorder_file 的 ancestor 链爬升"），`useReorder` 乐观更新时先本地校验再 invalidate。此为**有意的双语双份逻辑**，保证乐观 UI 与服务端裁决一致。

### 5.3 成绩单按模块学分校验规则

**目标**：教务成绩单（pdf.js 客户端解析出的 `TranscriptRecord[]`）→ 通识选修各模块达标判定。**全客户端、PDF 不上传服务器**。

**前端** `credits/lib/rules.ts::buildReport(records) -> CreditReport`（`:29-96`）算法：

```
1. 按 record.module 分组 → byModule
2. 展示模块集 = REQUIRED_MODULES ∪ 成绩单出现的模块（去重排序）
3. FOR 每个模块 m:
     earnedCredits ← sumCredits(recs)          // 浮点求和后 round(×100)/100，消 0.1+0.2 误差
     isRequired    ← m ∈ REQUIRED_MODULES
     minCredits    ← isRequired ? MODULE_MIN_CREDITS : 0
     meetsMin      ← earnedCredits >= minCredits
     // 子规则（四史/美育/劳动等）：按关键词匹配课程名
     specials ← SPECIAL_RULES.filter(module==m).map(s ⇒ {
         matchedCredits ← sumCredits(recs.filter(course.includes(s.keyword)))
         passed ← matchedCredits >= s.minCredits })
     module.passed ← meetsMin AND specials.every(passed)
4. 汇总 failedReasons（模块欠学分 / 子规则欠学分，各附 fmtCredit 明细）
5. RETURN { records, modules, passed: failedReasons.length===0, failedReasons }
```

**关键点**：`sumCredits` 用 `Math.round(s*100)/100` 消浮点误差；`fmtCredit` 整数去小数、非整保 1 位；子规则（四史/美育/劳动）用**课程名关键词包含**匹配再累加子学分——一门课可同时计入模块总学分与子规则学分。判定顺序：模块达标 = 模块最低学分满足 **且** 所有子规则满足。

### 5.4 拼音模糊索引

**目标**：导师中文名的拉丁化模糊搜索。schools.sqlite 由外部 supervisor-claw 只读导出、无拼音列且引擎 `mode=ro/immutable`，无法运行时加列，故建**内存索引**。

**实现** `services/schools_pinyin.py`：

```
类 PinyinIndex（__slots__=("entries")）：entries=[(advisor_id, full_pinyin, initials)]（全小写）
  build(rows):  FOR (id, name_cn):  full←"".join(lazy_pinyin(name)).lower()
                                     initials←"".join(lazy_pinyin(name, FIRST_LETTER)).lower()
  match(q):     ql←q.lower().去空格
                IF ql=="" OR 不含拉丁字母([a-zA-Z]) THEN RETURN ∅     // 纯 CJK 短路
                RETURN {aid | ql ⊂ full OR ql ⊂ initials}            // 双模式子串
```

- **两种匹配**（均子串=模糊）：全拼（"yaoming" 命中 姚明轩）与首字母缩写（"ymx" 命中 姚明轩）。
- **mtime 缓存重建**（`:64-76`）：模块级 `_cache_mtime`/`_cache_index`，`get_pinyin_index(session, mtime)` 仅当 mtime 变化才重扫 `SELECT id, name_cn FROM advisor` 重建——与只读引擎的 mtime 热重载**无显式失效即锁步**（claw 原子 rename → mtime 跳变 → 索引下次自动重建）。
- **纯 CJK 查询短路**返回空集，交由 `schools_query._build_where` 的 FTS5 MATCH + 转义 LIKE 处理；拼音命中的 `advisor_id` 集合再 OR 合并进 WHERE。
