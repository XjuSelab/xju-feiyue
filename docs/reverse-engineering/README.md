# xju-feiyue 逆向工程文档集（软件生存周期八阶段还原）

> 本目录是对已上线全栈平台**新疆大学飞跃手册（xju-feiyue / 内部代号 Aurash / 站名"费阅"）**的逆向工程产物：从低抽象层次的程序代码，反向推导并还原高抽象层次的系统设计，按软件生存周期"八个阶段"补齐符合中国软件工程国家标准（参照 GB/T 8567-2006《计算机软件文档编制规范》、GB/T 9385-2008《软件需求规格说明书》）的开发文书。

## 编制方法

采用**多智能体逆向工程**流程，两阶段完成：

1. **结构还原**：7 个专项分析智能体分别深读后端 API/数据/服务层、前端核心/feature 层、集成管线、文档基线，各自产出结构化分析；再由综合智能体整合为 [`00-架构逆向总览.md`](./00-架构逆向总览.md)。
2. **文书生成**：8 个文档智能体各自以架构总览为基线、并回读所负责阶段的真实源码（详细设计读服务类、测试读 `backend/app/tests/`、运维读 `deploy.sh`/`settings.py` 等），产出下列八份标准文书。

**一切结论以代码为准。** 逆向过程发现仓库内 `BACKEND_SPEC.md`、`INTEGRATION_REPORT.md`、`docs/architecture.md`、`backend/README.md`、`frontend/README.md` 均已严重陈旧（停留在"19 路由 / 全 mock / v1"阶段，实际已演进出 60+ 路由与 6 大 feature 纵切）——凡文档与代码冲突处，一律采信代码，详见架构总览第 8 节。

## 系统一句话画像

FastAPI 单体后端（`backend/app/`，routes→services→db 三层 + DI）前挂**三套物理隔离存储域**（可写主库 `labnotes.db` + 两个只读热重载 SQLite：schools/conferences）；React 18 SPA 前端（`frontend/src/`，features 纵切 + TanStack/Zustand + Zod 边界校验 + 唯一 fetch 封装）；辅以浏览器扩展（教务成绩单跨源中转）、内容离线管线（Notion/课程 wiki → markdown → 入库）、以及基于 Hugging Face 私有 Dataset 的状态同步与灾备。

- **测试/运维视角**：07 → 08（如何验证与如何上线维护）。
