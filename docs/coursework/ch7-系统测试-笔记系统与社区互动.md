# 七、系统测试（增量模块：笔记系统与社区互动）

> 负责模块：笔记系统（写作 + 展示）+ 社区互动（评论、表态、收藏、举报、拉黑、等级）+ 笔记合集
> 撰写者：赵文彪

## 1. 引言

本章对笔记系统与社区互动模块进行系统测试。测试范围以当前代码已实现的草稿发布、浏览、评论、赞踩、收藏、合集、签到与经验功能为主；FR-15～FR-18（举报、AI 审核裁决、拉黑）当前仅有部分数据模型或规划描述，不能作为已交付功能验收。测试环境为后端 pytest + httpx.AsyncClient + 内存 SQLite；AI 服务调用采用 dry-run 零联网模式。

## 4．对软件功能的结论

### 4.1 笔记表态（赞/踩/收藏）

**4.1.1 能力**

点赞、点踩、收藏接口均实现幂等与互斥逻辑。点赞与点踩为互斥关系（执行其一自动取消另一），重复操作不产生副作用。收藏独立于表态，支持幂等添加与取消。

**4.1.2 限制**

踩计数当前不在普通前端界面公开展示。当前代码不因点赞、点踩或收藏产生经验变动，因此不存在取消操作的经验扣回。

### 4.2 楼中楼评论

**4.2.1 能力**

后端评论支持两层嵌套结构（顶层评论 + 楼内回复），三层及以上嵌套被服务端校验拦截。删除顶层评论时数据库外键会级联清除其下回复。评论可携带锚点引用、@ 提及及最多 9 张图片 URL；当前前端主要接入平铺评论、锚点引用、创建与删除。

**4.2.2 限制**

评论列表接口当前按 created_at/id 倒序返回平铺列表，并用 cursor 做 keyset 分页；前端尚未把 parentId/replyTo/images/likes/dislikes 渲染为楼中楼树、图片九宫格或评论赞踩按钮。

### 4.3 合集管理

**4.3.1 能力**

后端合集支持创建、编辑、删除及笔记加入/移出。加入合集时执行三重校验：①操作者为合集创建者；②目标笔记为本人已发布且状态 visible 的笔记；③笔记未被其他合集收录（note_id 唯一约束）。合集上下文接口可查询同合集全部可见笔记并按 sort_order 排序；前端合集管理与详情页侧栏尚未接入。

**4.3.2 限制**

合集条目的排序当前支持创建/加入时传入或更新 sort_order，但尚未提供前端拖拽重排界面。

### 4.4 签到与经验

**4.4.1 能力**

签到以（用户，日期）复合主键保证每日至多一次，重复签到幂等返回不额外发放经验。经验以 xp_events 流水表记录每次增减，支持用户查询个人经验明细。

**4.4.2 限制**

连续签到天数计算已实现基础逻辑，当前经验发放固定为每日首次签到 +5，未实现连续签到 bonus 加成。等级徽章前端展示尚未对接。

## 5．分析摘要

### 5.1 能力

经代码 review 与现有测试核对，本模块中草稿发布、浏览、评论、赞踩、收藏、合集、签到与经验等功能已实现或可验证；举报、AI 审核裁决、拉黑仍停留在数据模型/规划层，不能写作已完成：

- 增量一核心链路（草稿发布、AI 润色/摘要、信息流、锚点评论、点赞）全部通过测试；
- 增量二扩展功能中，后端收藏、点踩、楼中楼创建约束、签到经验、合集管理均已通过测试；部分前端入口尚未接入；
- 幂等设计中，点赞、点踩、收藏、签到已以数据库约束或重复调用测试验证通过；举报幂等尚未实现接口验证；
- 互斥逻辑（赞踩互斥、合集单归属）均以测试用例覆盖。

### 5.2 缺陷和限制

| 编号 | 缺陷/限制 | 影响范围 | 建议 |
|---|---|---|---|
| D-01 | 评论图片上传与展示前端组件尚未完成集成 | 暂无法从 UI 为评论添加或查看图片九宫格 | 跟进前端图片上传和图片列表展示组件 |
| D-02 | 前端尚未渲染楼中楼树与评论表态按钮 | 后端 parentId/replyTo/comment_reactions 能力无法在页面完整体现 | 扩展 CommentSection schema 与 UI |
| D-03 | 合集条目不支持拖拽重排 | 合集内笔记顺序调整不便 | 增加 sort_order 批量更新接口 |
| D-04 | 合集管理、收藏列表、签到入口与等级徽章前端展示未对接 | 后端接口已具备但个人中心页面体验不完整 | 对接 `/auth/me/favorites`、`/collections/mine`、签到和 level badge 组件 |
| D-05 | 点赞/点踩快速切换未触发防刷限频 | 高频脚本可能制造无意义互动记录 | 增加操作冷却时间或用户级限频 |
| D-06 | 举报、AI 审查、人工裁决、拉黑接口尚未实现 | FR-15 至 FR-18 无法作为已交付功能验收 | 补齐 `/reports`、`/blocks`、管理端工单接口及前端入口 |

### 5.3 建议

1. 在系统上线前补齐评论图片、楼中楼树、评论赞踩、收藏/合集/签到入口与等级徽章的前端接入；
2. 为合集条目增加拖拽排序能力，提升创作者体验；
3. 为高频互动操作（点赞/点踩/收藏）增加用户级操作频次限制，防止脚本刷互动；
4. 后续实现举报、AI 审查、人工裁决与拉黑接口后，再补充对应验收测试和失败重试机制。
