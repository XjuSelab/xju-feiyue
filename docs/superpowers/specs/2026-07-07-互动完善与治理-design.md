## 阶段① 详细设计：高频互动防刷限流

### 目标
挡住脚本对点赞/点踩/收藏/评论表态/发评论的狂刷，正常用户（含手速连点）无感。

### 方案：进程内滑动窗口日志（sliding-window log）
贴合单机单 worker（nginx + systemd 一个 uvicorn）部署，零新依赖、O(1) 均摊、易测。多机/多 worker 时把存储换 Redis，接口不变。

### 组件与边界
- **新文件 `backend/app/ratelimit.py`**（与 `deps.py`/`settings.py` 同级，沿用扁平模块风格）：
  - `LIMITS: dict[str, tuple[int, float]]` —— 桶 → (最大次数, 窗口秒)。默认：
    - `"interaction"`: `(30, 10)` —— 每人 10 秒内点赞/点踩/收藏/评论表态合计 ≤ 30；
    - `"comment"`: `(10, 60)` —— 每人 60 秒内发评论 ≤ 10。
  - `class SlidingWindowLimiter`：`_hits: dict[str, deque[float]]`；`check(key, limit, window, now) -> (allowed, retry_after)`（清窗外时间戳；未满→记录当前、放行；已满→不记录、拒绝、`retry_after = 最早时间戳 + 窗口 - now`）；`reset()`。
  - 模块级单例 `limiter`。
  - `def rate_limit(bucket)` → 返回 FastAPI 依赖 `_dep(user=Depends(get_current_user))`：若 `settings.rate_limit_enabled`，读 `LIMITS[bucket]` 调 `limiter.check(f"{bucket}:{user.sid}", ...)`；超限抛 `HTTPException(429, "操作太频繁，请稍后再试", headers={"Retry-After": str(ceil(retry_after))})`。
- **`backend/app/settings.py`**：加 `rate_limit_enabled: bool = True`（env `RATE_LIMIT_ENABLED`）。
- **`backend/app/routes/interactions.py`**：给这些端点加依赖 `_: None = Depends(rate_limit("interaction"))`：
  notes 的 like/unlike/dislike/undislike/favorite/unfavorite，comments 的 like/dislike（含 DELETE）。
  给 `create_comment` 加 `Depends(rate_limit("comment"))`。
- **依赖复用**：`rate_limit` 依赖 `get_current_user` → FastAPI 单请求内缓存，鉴权只跑一次；未登录先被 401 挡下。
- **顺序**：限流在依赖解析阶段、进入处理函数前生效（先挡后干活）。

### 数据流
请求 → 鉴权(get_current_user) → rate_limit 依赖读桶配置 → 滑窗 check：放行则继续处理，超限则 429 + Retry-After。前端 `ApiError` 接住 429（阶段② toast "操作太频繁"）。

### 错误处理
- 未登录：401（早于限流）。
- 超限：429 + `Retry-After` 秒数。
- 限流器异常绝不阻断（防御式：check 内不抛业务无关异常）。

### 测试（`backend/app/tests/test_ratelimit.py`）
1. **单元**：直接测 `SlidingWindowLimiter`——放行 limit 次、第 limit+1 次拒绝且给出合理 retry_after；注入假 now 验证窗口滑出后恢复放行。
2. **集成**：monkeypatch `settings.rate_limit_enabled=True` 且把某桶 `LIMITS` 调小（如 `("comment",(3,60))`），连打超阈 → 断言 429 + 含 `Retry-After` 头；未超阈 → 正常 2xx。
3. **回归护栏**：`conftest.py` 里 `RATE_LIMIT_ENABLED=0` 默认关闭（避免既有多操作测试误触），并 autouse `limiter.reset()` 隔离用例。

### 验收
- 新测试全绿；既有后端测试仍全绿（`pytest`）。
- 手动：连点收藏 > 30 次/10s → 429 带 Retry-After。
